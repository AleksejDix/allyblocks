# Badge Component

A lightweight component for status indicators, labels, and highlighting information.

## Features

- Multiple visual variants (default, secondary, destructive, outline)
- Non-interactive by default - used primarily for visual indicators
- Can be rendered as different elements using `asChild`
- SVG icon support with appropriate styling

## Usage

```tsx
import { Badge } from "@/components/atoms/Badge";

// Default badge
<Badge>New</Badge>

// Secondary variant
<Badge variant="secondary">Beta</Badge>

// Destructive variant
<Badge variant="destructive">Error</Badge>

// Outline variant
<Badge variant="outline">Draft</Badge>

// With icon
<Badge>
  <Icon name="check" />
  Verified
</Badge>

// As a link (becomes interactive)
<Badge asChild>
  <a href="/status">Status</a>
</Badge>
```

## Props

The Badge component accepts the following props:

| Prop        | Type                                                     | Default     | Description                                                        |
| ----------- | -------------------------------------------------------- | ----------- | ------------------------------------------------------------------ |
| `variant`   | `'default' \| 'secondary' \| 'destructive' \| 'outline'` | `'default'` | Visual style variant                                               |
| `asChild`   | `boolean`                                                | `false`     | When true, the component will render its child as the root element |
| `className` | `string`                                                 | `undefined` | Additional CSS classes                                             |

Plus all standard props for a `<span>` element.

## File Structure

- `Badge.tsx`: Core component implementation
- `Badge.types.ts`: TypeScript type definitions
- `Badge.variants.ts`: Style variants using class-variance-authority
- `Badge.stories.tsx`: Storybook examples
- `index.ts`: Entry point for exports

## Accessibility

The badge component maintains appropriate color contrast ratios for readability. When used as an interactive element (via `asChild`), the host component should provide the necessary accessibility attributes.

## Customization

Extend or modify the appearance using the `className` prop:

```tsx
<Badge className="px-4 py-2 text-sm">Custom Size</Badge>
```
